    In main(), replace the single `try { await processCase(page, processes[0], flags) } finally { await browser.close() }` block with:

    1. Declare `let hadFailure = false` before the try/finally
    2. The outer try/finally wraps the ENTIRE browser session (browser is launched before try, closed in finally):
       ```
       try {
         for (const proc of processes) {
           try {
             await processCase(page, proc, flags);
           } catch (err) {
             console.error('[ERROR] ' + (proc.label ?? proc.caseNumber) + ': ' + err.message);
             hadFailure = true;
           }
           // delay between cases — but NOT after the last one
           if (proc !== processes[processes.length - 1]) {
             const delayMs = BASE_DELAY_MS + Math.floor(Math.random() * JITTER_MS);
             if (flags.verbose) console.log('[verbose] Waiting ' + delayMs + 'ms before next case...');
             await sleep(delayMs);
           }
         }
         process.exitCode = hadFailure ? 1 : 0;
       } finally {
         await browser.close();
       }
       ```
    3. The browser.close() in the finally block satisfies REL-02 — it runs even if every case throws.
    4. process.exitCode is set INSIDE the try block (after the loop) and BEFORE the finally executes — this is the correct order. Use process.exitCode (not process.exit()) to allow async finally to complete.

    Also add these verbose log lines to processCase() if not already present:
    - Before scrapeCase() call: if (flags.verbose) console.log('[verbose] Processing case:', proc.caseNumber, proc.label ? ('('+proc.label+')') : '');
    - After saveState() call: if (flags.verbose) console.log('[verbose] State saved to', path.join(STATE_DIR, proc.caseNumber + '.json'));

| Boundary | Description |
|----------|-------------|
| config/processes.json → main() loop | All caseNumber values processed in the loop; each must be validated before file path use |
| scrapeCase() return value → saveState() | Row data from live DOM written to disk; data is display-only, no execution risk |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | processCase() per-case isolation | mitigate | Each case wrapped in try/catch — a malformed case cannot poison state for other cases |
| T-02-02 | Availability | browser.close() on exception path | mitigate | try/finally wrapping entire browser session ensures browser.close() runs even on total failure (REL-02) |
| T-02-03 | Denial of Service | delay between cases | accept | BASE_DELAY_MS + jitter enforces polite sequential access; no parallel requests (CLAUDE.md constraint) |
| T-02-04 | Tampering | empty table data written to state | mitigate | REL-03 assertion (rows.length > 0) aborts before saveState() is called — prevents overwriting known state with empty array |
</threat_model>
